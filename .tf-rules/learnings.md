[chat] Metaobject content in snippets: shop.metaobjects.<type>.values is a LAZY DROP — .first and [0] index BOTH return a blank object (empty system.handle, empty fields) even when .values.size is correct. Inside the loop access fields directly (entry.field_key.value — NOT entry.fields.field_key, which is GraphQL-only and nil in Liquid). Assign hardcoded fallbacks BEFORE the loop and override inside with | default: so the card renders when no entry is published. Confirmed against the did_you_know_subscriptions_footer metaobject.
