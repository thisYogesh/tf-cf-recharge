[chat] Recharge Affinity slots are keyed by data-recharge-slot name in recharge-slot-injector.js (Map) AND by Recharge SDK — only ONE template can bind per slot name. To stack multiple widgets in one slot (e.g. *.sidebar), put them all inside a SINGLE <script type="text/html" data-recharge-slot="..."> template; a second template with the same slot name silently collides and one widget never renders.
[chat] Liquid inside a Recharge slot's <script type="text/html"> template still renders server-side first (it's a Liquid string baked into the DOM before Recharge injects it), so shop.metaobjects.* and other Liquid data resolve normally inside slot markup.
[chat] Metaobject content in snippets: read the singleton entry via shop.metaobjects.<type>.values.first, then access fields directly (entry.field_key.value — NOT entry.fields.field_key, which silently returns nil in Liquid). Always wrap with | default: to a hardcoded fallback so the card renders even when no entry is published or the definition lacks Online Store/storefront access.
